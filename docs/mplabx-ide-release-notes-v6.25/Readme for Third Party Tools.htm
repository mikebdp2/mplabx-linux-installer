1       Third Party Tools Support

MPLAB X IDE supports development tools from many different vendors. If you encounter problems using the tools with MPLAB X IDE, see the vendor website for more tool information. Also ensure you have the latest version of the tools and associated MPLAB X IDE plug-ins.

Plug-ins can be downloaded from the plug-in portal using the MPLAB X IDE plug-in manager or from the third-party vendors. See the MPLAB X IDE documentation for how to install a plug-in.

5       SEGGER J-Link and MPLAB X IDE

5.1      Update J-Link Library for Linux OS

Follow these steps to update the SEGGER J-Link library for MPLAB X IDE on Linux:

  1. On the SEGGER website, download the latest J-Link Software and Documentation pack. Select the latest version (for example V7.98h) and then click on the 64-bit DEB Installer to start the download.
  2. Extract the contents of the file JLink_Linux_V798h_x86_64.deb. Then in the folder opt/SEGGER/JLink_V798h, find the file libjlinkarm.so.7.98.8.
  3. Copy libjlinkarm.so.7.98.8 to the MPLAB X IDE install folder /opt/microchip/mplabx/v6.20/mplab_platform/thirdparty and rename it to libjlinkpic32.so.7.98.8.
  4. Create a symbolic link libjlinkpic32.so to point to libjlinkpic32.so.7.98.8.
    Note    : You must remove the existing symbolic link if it already exists.

5.2      Known Problems

XPI-53: [SEGGER J-Link] Breakpoints set in the data memory using SFRs with or without pass count do not halt the target.

XPI-49: [SEGGER J-Link] The MPLAB X IDE dashboard does not reflect the number of set breakpoints correctly when using SEGGER J-Link probe. Even when a breakpoint is used, all breakpoints are shown as available.

XPI-47: [SEGGER J-Link] Error messages reported using MPLAB X IDE and SEGGER plugin are cryptic.
