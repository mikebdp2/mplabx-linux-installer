Device Device supported in MPLAB® X IDE. Gen 4 and Gen 5 tools do not support debug/emulator headers.
Program Memory Reserved Device Program memory reserved for debug.
File Registers Reserved Device File Registers memory reserved for debug.
Program Memory Reserved
(SW BP Support)		 Device Program memory reserved for debug. Software Breakpoint support may require more resources for debug. Not all tools support software breakpoints.
File Registers Reserved
(SW BP Support)		 Device File Registers memory reserved for debug. Software Breakpoint support may require more resources for debug. Not all tools support software breakpoints.
MCLR Pin Reserved/Shared Master Clear (MCLR) pin may be reserved for debug and/or shared with Vpp. If reserved/shared, the pin cannot be used as digital I/O while debugging.
LVP Disabled Low voltage programming (LVP) will be disabled on devices that support this feature.
NOTE: In-circuit debuggers/emulators do not support Low Voltage Programming (LVP) and will disable this function.
You cannot enable it through programming using the debugger.
However, you can enable it through a dedicated programmer, such as MPLAB PM3.
Device Pins Reserved Programming pins (RB6/RB7, PGC/PGD) may be reserved for debug.
Emulation pins (EMUC/EMUD) may be reserved for debug.
Other functions multiplexed on these pins will not be available during debug.
Stack Reserved Some stack levels may be reserved for debug. If levels are reserved, the debugger will need to control the stack,
so on some device TOS registers (TOSL, TOSH, TOSU) will also be reserved.
