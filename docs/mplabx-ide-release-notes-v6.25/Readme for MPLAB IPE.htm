2       Other Configurations

Apple’s latest version of macOS, 10.15 Catalina, removes support for 32-bit applications. This means MPLAB X IDE, MPLAB XC16, and XC32 may not run on macOS Catalina. You will get a message indicating that the application needs to be updated.

Microchip is actively working to update the MPLAB X IDE, MPLAB XC16, and XC32 to be 64-bit applications. When the 64-bit versions are released, the toolchains will no longer run on 32-bit operating systems. We expect to release 64-bit versions of the IDE, MPLAB XC16, and MPLAB XC32 within the first half of 2020.

To run 32-bit applications, Apple unofficially recommends either that you keep an old Mac on hand that runs a pre-Catalina version of macOS or that you partition your current Mac so that it can start up with an older macOS version as well as Catalina. Another option may be to run an older macOS version in a virtual machine using VirtualBox, Parallels Desktop, or VMware Fusion.”

 

MPLAB X IDE is no longer supported on Windows XP. This means MPLAB X IDE will no longer be tested on Windows XP and JRE8 will be the new default JRE. To continue using Windows XP, JRE 7 will need to be installed. The MPLAB X IDE installer will search for JRE 7 on your system, and if it is not found will recommend sites for download.

See the Readme for MPLAB X IDE.htm for other support information and system requirements.

8       Important Notes

8.1    Switcher Use

If switching from MPLAB IDE v8.xx, you must use the Switcher to switch the PM3 drivers.

If MPLAB IDE v8.xx or earlier is already installed on your computer, you need to run the Switcher program to switch from MPLAB IDE v8 drivers to MPLAB X IDE drivers, which are installed the first time your switch.

The Switcher program is a GUI application called MPDDSwitch (32-bit operating system) or MPDDSwitch64 (64-bit operating system). This should be available as the desktop icon MPLAB Driver Switcher.

For more information on the Switcher program, refer to the MPLAB X IDE online help.

8.2    Checksums

The following details changes to checksum calculation for the MPLAB IPE.

In addition, see the section “How Do I Stop My Project’s checksum from Changing? In the “How To’s” chapter of your MPLAB XC C Compiler User’s Guide.

8.2.1   Checksum Changes

For new devices, CRC32 checksums will be provided upon file import. For legacy devices, nothing will change.

There will no longer be 'read' checksums for the new devices. You will need to perform a separate 'Verify' to verify the code.

About CRC32 Checksums

For new devices, MPLAB X IDE and MPLAB IPE run a 32-bit CRC calculation on the entire hex file to find its checksum. The checksum uses the standard CRC32 algorithm with the polynomial 0x4C11DB7:
(x32 + x26+ x23 + x22 + x16 + x12 + x11 + x10 + x8 + x7 + x5 + x4 + x2 + x + 1)

CRC32 Table Generation Reference:

http://www.sunshine2k.de/coding/javascript/crc/crc_js.html

Crosscheck the MPLAB X IDE CRC32 calculation using this website:

https://md5calc.com/hash/crc32

 

8.3    SST Memory Devices

SST memory devices with large memories (>1MB) will take time for the hex file image to import.  Please allow the time needed for the import to occur.

8.4    Blank Check

In version 2.25 for dsPIC33EV devices, the Blank Check function will check all memory areas including the USER OTP memory.  Future versions of the blank check operation will separate out the USER OTP memory section from all other sections.

The PIC24FJ128GA204 family of devices fails Blank Check upon Erase, although the devices are actually blank. This is due to a change in the programming specification for these devices for Configuration Word 2 <bit 11>. Ignore the failure of the blank check upon erase for these devices when using the MPLAB ICD 3 In-Circuit Debugger, MPLAB REAL ICE in-Circuit Emulator, MPLAB PICkit 3 Debugger/Programmer or MPLAB PM3 Programmer.

8.5    Timestamp

The Output window displays a timestamp for events. For example,

2015-02-19T11:02:41-0700- Completed loading IPE.

This indicates the date (year-month-day), time (hour:minute:second), and the GMT (Greenwich Mean Time) plus or minus offset.

8.6    SQTP Files

SQTP files for Flash Data memory generated prior to MPLAB IPE v2.20 will work with MPLAB PM3 firmware up to v3.00. SQTP file generation for Flash Data memory has been modified to the 32-bit byte order (12345678) and the firmware is updated accordingly. A new SQTP file must be regenerated to work with MPLAB IPE v3.00.

For PIC32 devices only, MPLAB IPE v3.15 provides a dialog box to choose either 32-bit byte order (12345678) or 16-bit byte order (56781234) when loading an SQTP file.

The MPLAB PM3 does not support SQTP for devices with a RAM size of more than 1 MB of memory. Devices affected include PIC32MX2048EFH100 and others that have more than 1MB of memory.

8.7    Saving Split Operating Systems to the PM3 SD Card

In production houses, there may be a need to program several different device family architectures without having access to a computer.

Between MPLAB IPE v3.25 – v3.40 the PM3 operating system was split into two parts based on the device architecture (32-bit vs all other devices). Now with MPLAB IPE v3.40, the PM3 operating system and database are split into three parts based on the architectures for 8-, 16- and 32-bit devices.

MPLAB IPE v3.40 includes the feature to save the three PM3 operating system firmwares supporting each of the device family architectures into a PM3 SD card.

When the PM3 environment is selected in Stand-Alone mode, the MPLAB PM3 programmer will load the correct operating system and database from the PM3 SD Card to support the device.

This new feature is available only in the MPLAB IPE v3.40, not in the MPLAB X IDE. The MPLAB X IDE will download one of the split PM3 operating systems based on the architecture of the device chosen in the project when connected to a computer.  To download all three operating systems to the PM3 SD card to allow for complete stand-alone mode, please use the MPLAB IPE v3.40 (or greater).

To save all three operating systems into the PM3 SD card:

Insert any supported SD card into PM3 SD card slot.

Open MPLAB IPE v3.40 or greater and connect the MPLAB PM3 programmer to the computer.

Select Settings >Save firmware into SD Card.

On the MPLAB PM3 unit, select a PM3 environment to open. 

Once opened, the PM3 will automatically download the correct PM3 operating system split based on the device saved in the PM3 environment.

Points to Be Considered:

Please ensure the PM3 operating system firmware version stored on the SD card matches the operating system firmware version on the MPLAB PM3 programmer. A mismatch may occur if an upgrade of the PM3 operating system was performed but the SD card firmware was not updated. Use the MPLAB IPE v3.40 or greater to select the Settings>Save firmware into SD Card option to load the PM3 SD card with the desired PM3 operating system firmware version.

Any firmware versions of v3.40 or greater are not compatible with earlier versions of MPLAB IPE. It is highly recommended that the MPLAB PM3 use the firmware packaged within the same MPLAB IPE version.

Prior to MPLAB IPE v3.25

One operating system supported

MPLAB IPE v3.25-3.35

Two operating systems supported (32-bit and all other devices)

MPLAB IPE v3.40 or greater

Three operating systems supported (8-, 16-, 32-bit devices)

 

8.8    Vdd_App and Vdd_Nom

In the power option for PM3, MPLAB 8 uses Vdd_App for Verification and Vdd_Nom is used for all other operation (program, read, erase, blank check). But in MPLAB X IPE, it is the other way around (Vdd_Nom is used for verification and Vdd_App is used for all other operation). Note that in a future release this difference will be removed and MPLAB X IPE will be modified to match with MPLAB v8.

8.9    dsPIC30F5013 RBS Configuration Bits

The MPLAB ICD 3, MPLAB PM3 and PICkit 3 fail to program the dsPIC30F5013 RBS configuration bits for small-sized Boot RAM and Medium-sized Boot RAM when using the MPLAB IPE.

8.13 Verifying Device Utilizing Memory Ranges

When utilizing memory ranges and performing a verification of the device on an MPLAB ICD 3, PICkit 3, or REAL ICE, the entire device program memory and configuration sections will be verified, not just the selected memory ranges. In the next generation tools (MPLAB ICD 4, MPLAB PICkit 4, etc.), when a memory range is selected and a verification is performed, only the specified range will be verified.

8.14 For AVR device with UPDI using the MPLAB PICkit 4

On low pin count AVR devices with UPDI, the UPDI pin can be configured as GPIO or RESET by configuring the RSTPINCFG configuration bits. To do further programming, the MPLAB PICkit 4 will have to use a high voltage pulse to reactivate the UPDI interface. When using the high voltage pulse, you must make sure that all circuits connected to the UPDI wire can tolerate a pulse of at least 12V.

In Project Properties, go to the “PICkit 4” category, “Communications” option category, “High Voltage Activation Mode”.

Drop-down menu item

Description

No High Voltage

Default setting.

Simple High Voltage Pulse

The tool will try to activate the interface by issuing a high voltage pulse. This procedure is safe if the pin is configured as an input.

User Power Toggle

In this mode the user will be prompted to toggle power on the target device. Once the tool detects that the power returns it will issue a high voltage pulse before the target device pin is configured, making the activation procedure as gentle as possible.

 

GPIO vs. UPDI Operation:

When using a high voltage pulse to reactivate the UPDI interface, the reactivation is only temporary, but it will retain the UPDI functionality until the next reset. After the next reset, the pin will go back to the configuration as specified by the RSTPINCFG configuration bits. To have the pin configured as UPDI after a reset, the user will have to change the RSTPINCFG configuration bits back to UPDI.

It is possible to perform a debug session when the RSTPINCFG is configured to GPIO, but the pin will be temporarily configured as UPDI, and the pin will not operate as a GPIO pin.

8.15 Programming Configuration Bits of AVR and SAM Devices in MPLAB IPE

To program the configuration bits of AVR and SAM devices in MPLAB IPE, they need to be part of the hex file only. This differs from the MPLAB X IDE, where they can be programmed separately via the configuration bits memory window.

For PIC devices, however, the configuration bits can be programmed separately using MPLAB X IDE and MPLAB IPE.

8.16 DFPs and Tools Pack Support

·         DFP support is available for all devices.

·         Tool Pack support is available only for MPLAB ICD 4, PICkit 4 and MPLAB Snap.

·         The MPLAB X IDE and MPLAB IPE share the same packs. Any changes in packs using the pack manager will affect both the IDE and IPE.

·         Packs can be installed/ uninstalled using Pack Manager. The Pack Manager GUI is available only in MPLAB X IDE, therefore, the MPLAB X IDE must be installed to be able to use the Pack Manager. Refer to the MPLAB X IDE online help – Additional Tasks>Work with Packs – for more information.

·         User can a select specific version Device Family Pack (DFP) and Tool Pack version in the MPLAB IPE under the Advanced Mode, Settings tab. By default, IPE uses latest available pack. Note, in order to have additional packs available, the Pack Manager in MPLAB X IDE must be used to add packs.

8.17 Importing Hex Files

Hex files, including eep files, can be imported into MPLAB IPE using the File>Import>Hex or File>Import>Multiple Hex options. These options accept hex and other types of hex files such as eep files. The eep files are hex files that have a non-hex extension.
